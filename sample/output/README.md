## Quality checklist (what makes this a good PRD)

- One Thing stated up front, mirrored from user's own words
- Chosen approach + backup route (with honesty-ladder footprint)
- Platform, constraints, and cost all expressed as ranges with source tiers
- Feature count = 5 (within ≤7 cap)
- Inferred values tagged `[inferred]` / `[assumed]`
- Open questions listed explicitly (not buried as hedging)
- Sign-off block with 6-char verification code
- Cites specific Articles (e.g. Art 3, Art 4a) so the reader can audit which rule applies where
- Property labels: [SPEED-FIRST], [COST-FIRST], etc.
